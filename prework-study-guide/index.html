HTML

  • Hypertext Markup Language (HTML): create the structure of a webpage, creartes a basic framwork.
  • "TITLE": defines the title page in "H1" (Important for SEO: Search Engine Optimization).
  • The "head" element contains information or metadata about the webpage.
  • "Meta" is data about or web page, the browser uses, to be effective for webpage users.
  • The "body (semantic HTML) element" represents the visible content shown to the user.
  • "Element" entire element from opening tag to closing tag
  • "Tag": only appears inside the angle brackets.
  • "Semantic HTML element": name of the element describes its content and the role it plays on a webpage.
  • Not all elements are semantic, though. It is still common to see non-semantic elements such as
    to create a division in the page, or to create an inline division

Git

  • git status: checks what branch we are currently on
  • git checkout -b "branch-name": "master/main" or "branch name": creates a new branch and switches to it
  • git stash save: create stash that contains your changes, but it isn't associated with any commit or even branch
  • git add -a: add new files to local git repository
  • git commit -m: message such as "added code to HTML file"
  • git pull: pull request, or "PR", is used to merge a comparison branch (in our case, the feature branch) into the base branch.
  • git push" ": push up all of your changes to the main branch
  • git init: To initialize a repository, Git creates a hidden directory called .git
  • git clone: "paste copied SSH value"
